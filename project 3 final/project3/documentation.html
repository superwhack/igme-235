Bee Morse Project 3 Documentation: javasqueebs!

Process:

I did a lot of the architecture at first, trying to keep everything as abstract as possible. I only added what I thought I needed, but I had most of the classes laid out in my head in the beginning. I also made sure to use lots of classes in my html to make the later beautification easy. I definitely over complicated certain aspects, if I were doing this again I would definitely simplify the collection/materials code, as it led to some unnecessary bugs. I wish I could’ve implemented a save/load system as well. However, I am very proud with how this came out. It is abstract enough that I could easily add new features or reuse aspects of this for other projects.

Sources:

I didn’t use any tutorials. I discovered how to use requestAnimationFrame() and setTimeout() using the Mozilla Developer Documentation and a bit of W3Schools.

Whats cool

I think its cool how abstract I made everything, allowing for expansion and modifications easily. If I choose to return to this project, development would be quite quick. I’m quite proud of how the progress bars came out, as well as the cheating features I implemented. I like the art that I did, and I think the game has a fun progression to it. I’m proud of how the inventories automatically update when you gain new materials or squeebs, regardless of what screen you’re on. Im proud of how the drop-down automatically populates whenever you click it, and how the adventure system works as a whole.

Meeting Requirements:

Functional:

I used javascript with the browser DOM, made a fun game that operates intuitively I used ideas we never went over in class, and my program throws no errors.

Design:

I made a simplistic but clear visual interface, with clear labels and a basic tutorial of how everything works. User errors cannot break the program. Items squash, stretch, and adapt to different screen sizes.

Media:

My art is properly scaled, and are all PNGs. Sound notifies you of a new squeeb. I added fonts from Google Open Fonts.

HTML/CSS:

My HTML is reasonably valid and semantic. My CSS is external and valid.

Code Conventions:

I have a lot of ES6 classes, only use let, and didn’t use old methods of DOM traversal. I used lots of functions and classes to avoid repeating myself. My javascript is separated into 3 files. Variables and functions start with a lowercase letter. Nothing is thrown in the console, and my code is commented.
